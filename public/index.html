嗨！我是重年，在網頁開發領域有一個約定俗成的詞彙，專門用來形容不仰賴開發工具與框架，只使用原生語法來開發網站的風格，這個詞彙是 Vanilla。

例如，當開發者說使用 Vanilla JavaScript 時，意思是只用 JavaScript 的原生語法，而不借助像 jQuery、React、Angular 或 Vue.js 這類函式庫或框架。同樣地，Vanilla CSS 指的就是直接使用標準的 CSS 語法，而不依賴像是 Bootstrap 或 Tailwind CSS 等 CSS 框架。

而這個網站是 GitHub 上 Plain Vanilla 專案（連結）的本地化，是用繁體中文講解如何在不仰賴開發工具與框架的情況下，只使用 HTML、CSS 和 JavaScript 的原生語法來開發網站。內容會對 Vibe Coding 很有幫助，而且作者有在 issue 中（連結）明確開放，所以就著手進行了翻譯。

本站更新於 2025 年 5 月 19 日，本地化進度 5 %，即時資訊建議請參考原專案的官方網站（連結）。

關於網站

這裡概述了在不使用建置工具或框架的情況下，僅憑一個編輯器、一個瀏覽器以及網頁標準，來製作網站和網頁應用程式的主要技術，包含以下主題：

如何建立元件
使用 Web Components 作為基礎建構模組，透過 HTML、CSS 和 JavaScript 的原生語法，來打造更高級別的原始元素，藉此取代 React 或 Vue 等框架。
如何設定樣式
完整發揮現代 CSS 的強大功能，來取代 CSS Modules、PostCSS 或 SASS 等工具所提供的便利性。
製作一般網站
基於 Web Components 建立網頁專案，並在無需建置工具、框架或伺服器端邏輯的情況下，將其部署至正式環境。
製作互動網站
如何運用原生語法建構單頁式網頁應用程式 (SPA)，以及如何處理路由和狀態管理。

這個教學專為熟悉 HTML、CSS 及 JavaScript 的開發者設計，並不適合初學者瀏覽。如果您剛開始學習網頁開發，正在尋找入門資源，建議先從 The Odin Project 或 MDN 的 學習 Web 開發 開始入門。

網站緣起

現代網頁開發框架功能強大，能協助開發者快速建構出內容豐富、結構完善的網頁應用程式，因此非常值得投入時間學習。不過框架的豐富功能也伴隨著複雜性，而且為了確保安全性，開發出來的專案往往需要定期維護並保持更新。

但 Vanilla 風格的網頁則另闢蹊徑，因為目前的瀏覽器環境已經對網頁標準提供了絕佳的支援，所以它選擇犧牲部分短期便利，以換取如架構簡潔、幾乎無需維護等長期優勢。
